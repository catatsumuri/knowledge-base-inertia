# Gitの操作

## 概要

Gitリポジトリで作業する際、Kiro CLIのファジーファインダーはGitと統合されており、コンテキストに関連ファイルを効率よく追加できます。

この機能により、Git管理下のファイルを素早く特定し、必要なものだけを取り込むことが可能です。

---

## Git対応ファジーファインダーの仕組み

Git情報と自動的に連携し、強化されたファイル選択機能を提供します。

### 主な特徴

* Gitで追跡されているファイルを自動認識
* ファイルごとにGitステータスのインジケーターを表示
* Git履歴に基づき、関連性の高いファイルを優先表示

---

## Git対応ファイル選択の使用方法

### 1. リポジトリへ移動

ターミナルで対象のGitリポジトリへ移動します。

```bash
cd your-repository
```

### 2. コンテキスト追加コマンドを実行

```bash
/context add
```

### 3. ファジーファインダーの操作

インターフェース上に、Gitステータス付きでファイル一覧が表示されます。

#### Gitステータス表示

| 表示 | 意味       |
| -- | -------- |
| M  | 変更済みファイル |
| A  | 追加済みファイル |
| ?  | 未追跡ファイル  |

### 4. ファイルの絞り込みと選択

* キーワードを入力して絞り込み
* Gitステータスを検索条件として利用可能
* 矢印キーで移動
* `Enter` キーで選択ファイルをコンテキストへ追加

---

## Git対応選択のヒント

* Gitステータスを活用すると、変更済み・未追跡ファイルを素早く特定できる
* 最近変更されたファイルは優先表示されやすい
* `.gitignore` によって無視されているファイルも表示可能だが、優先度は低い

---

## まとめ

Kiro CLIのGit統合ファジーファインダーを活用することで、作業中の変更や追加ファイルを即座にコンテキストへ反映できます。

Git履歴とステータス情報を軸にした選択機能により、効率的なワークフローを実現できます。
